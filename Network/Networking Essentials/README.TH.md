## Networking Essentials

### **1. DHCP (Dynamic Host Configuration Protocol)**

**หน้าที่:**  
แจกจ่าย IP address และค่าการตั้งค่าเครือข่ายให้กับอุปกรณ์โดยอัตโนมัติ

**กระบวนการทำงาน:**

1. **Discover** – เครื่องลูกข่าย (Client) ส่งคำขอ IP ออกไป
    
2. **Offer** – เครื่องเซิร์ฟเวอร์ (DHCP Server) เสนอ IP ที่ว่างให้
    
3. **Request** – Client ขอใช้ IP นั้น
    
4. **Acknowledge** – Server ยืนยันการเช่า (Lease)
    

**ตัวอย่าง:**  
เมื่อเชื่อมต่อ Wi-Fi ที่บ้าน ระบบ DHCP จะให้ IP โดยอัตโนมัติ เช่น `192.168.1.23`

---

### **2. ARP (Address Resolution Protocol)**

**หน้าที่:**  
จับคู่ IP address กับ MAC address ของอุปกรณ์ในเครือข่ายเดียวกัน

**หลักการทำงาน:**

- เครื่อง A ส่งคำถาม “ใครคือ 192.168.1.1?”
    
- เครื่อง B (ที่เป็นเจ้าของ IP นั้น) ตอบกลับด้วย MAC address ของตน
    
- เครื่อง A จะเก็บข้อมูลนี้ไว้ใน **ARP cache** เพื่อใช้ครั้งต่อไป
    

**คำสั่งดูตาราง ARP:**

`arp -a`

---

### **3. NAT (Network Address Translation)**

**หน้าที่:**  
ทำให้หลายอุปกรณ์ในเครือข่ายภายใน (Private IP) สามารถออกอินเทอร์เน็ตผ่าน IP สาธารณะเดียวกันได้

**วิธีทำงาน:**

- เครื่องในเครือข่ายภายในส่งข้อมูลออกทาง Router
    
- Router จะแปลง Private IP ให้เป็น Public IP ก่อนออกเน็ต
    
- เมื่อข้อมูลตอบกลับ Router จะส่งกลับให้เครื่องต้นทางที่ถูกต้อง
    

**เปรียบเทียบง่าย ๆ:**  
เหมือนพนักงานต้อนรับในบริษัทที่ใช้เบอร์โทรหลักเดียวกันรับสายแทนพนักงานหลายคน

---

### **4. ICMP (Internet Control Message Protocol)**

**หน้าที่:**  
ใช้ในการส่งข้อความแจ้งข้อผิดพลาด หรือข้อมูลการทำงานระหว่างอุปกรณ์ในเครือข่าย

**เครื่องมือที่ใช้ ICMP:**

- **Ping** → ตรวจสอบการเชื่อมต่อ
    
- **Traceroute** → ตรวจเส้นทางของข้อมูล
    

**ประเภทข้อความตัวอย่าง:**

- Echo Request/Reply (ใช้ใน ping)
    
- Destination Unreachable (ปลายทางไม่สามารถเข้าถึงได้)
    
- Time Exceeded (หมดเวลาในการส่งต่อ)
    

---

### **5. Ping และ Traceroute**

**Ping:**

- ใช้โปรโตคอล ICMP
    
- ตรวจสอบว่าเครื่องปลายทาง “ออนไลน์หรือไม่”
    
- แสดงเวลาที่ใช้ในการส่งข้อมูลไป-กลับ (RTT)
    

`ping google.com`

**Traceroute:**

- แสดงเส้นทางที่แพ็กเกตเดินทางผ่าน Router ต่าง ๆ
    
- ใช้ช่วยตรวจหาว่าข้อมูลติดอยู่ตรงไหน
    

`traceroute google.com`

(บน Windows ใช้คำสั่ง `tracert google.com`)
